The object manager uses abstraction-implementation mappings when the constructor signature of a class requests an object by its interface. The object manager uses these mappings to determine what the default implementation is for that class for a particular scope.
If you want to override a public or protected method from a core class, utilize the preference node from di.xml to achieve it.
Predefined variables will take the following values:
${FOR}   Target PHP Class to change its concrete implementation
${TYPE}   PHP Class that will be instantiated instead of specified in predefined ${FOR} variable
